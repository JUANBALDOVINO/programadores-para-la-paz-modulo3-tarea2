Explicación del comportamiento del backend (Ruta GET http://localhost:3000/reportes):

Cuando se realiza una solicitud HTTP GET a la ruta 'http://localhost:3000/reportes', 
el servidor Node.js/Express procesa la petición enviando como respuesta un arreglo en formato JSON. 

Si previamente se envió un reporte mediante el método POST, el servidor responderá 
con una lista que contiene dicho objeto. Por ejemplo:

{
  "tipo": "Vialidad",
  "descripcion": "Bache profundo en el carril central"
}

Flujo del proceso:
1. El cliente (navegador, Postman o curl) solicita la información almacenada.
2. El servidor busca en su estructura de datos (por ejemplo, un arreglo en memoria o 
una base de datos) los registros guardados.
3. El servidor retorna un código de estado 200 OK junto con el arreglo de reportes, 
permitiendo visualizar de manera centralizada todas las incidencias registradas en el sistema.

Resultado:

HTTP/1.1 200 OK
X-Powered-By: Express
Content-Type: application/json; charset=utf-8
Content-Length: 80
ETag: W/"50-dycCOhoqB5Bxt9hVfF5beFaGQI8"
Date: Thu, 11 Jun 2026 17:19:49 GMT
Connection: close

[
  {
    "id": 1,
    "tipo": "Vialidad",
    "descripcion": "Bache profundo en el carril central"
  }
]
